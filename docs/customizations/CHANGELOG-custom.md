# 二开改动清单

记录每个二开改动:改了什么、为什么改、影响范围、对应分支/PR。
每个 feature/hotfix 合并后追加一条,上游升级时用此文件判断冲突保留策略。

## 格式

### [版本/日期] 改动标题
- 分支/PR: `feature/xxx` / #PR号
- 机器: <机器标识>
- 改动范围: <涉及模块/文件>
- 原因: <业务需求或问题>
- 影响上游合并: <是否会与上游改动冲突,及应对>
